Birthday Paradox

The Birthday Paradox explores the probability that in a group of 23 people, at least two share a birthday. This project simulates this probability using a series of experiments.

Functional Requirements: The program runs experiments for group sizes ranging from 5 to 100 (in increments of 5) with 20 trials per size, computing the probability of shared birthdays.

Developed in Java, the project consists of multiple classes including Simulation, Experiment, Trial, and Birthday. It utilizes a single instance of a random number generator for consistency and includes edge case testing for robustness. I focused primarily on writing the birthday, trial and test classes while also leading our group to ensure all of our code worked seamlessly.

Together, we implemented unit tests to ensure code correctness, focusing on edge cases and test coverage exceeding 80%. Used mocking techniques and error handling to prevent invalid method usage.

What I Learned: Improved Java programming, clean coding practices, debugging techniques, and Git workflow management including branches, pull requests, and code reviews.

Elevator Simulator

The Elevator Simulator is a Java-based console simulation that models the behavior of different types of elevators in a multi-story building. It processes elevator requests and simulates movement over multiple rounds using customizable elevator types and scheduling strategies.

Functional Requirements: The simulator accepts elevator calls, assigns them using either a Simple or Complex scheduler, and simulates elevator movement one floor per round. It supports Standard, Freight, Express, and Penthouse elevator types, each with distinct rules and capabilities.

Design: The project follows clean object-oriented design using abstraction, encapsulation, and polymorphism. The Elevator class is abstract, with concrete subclasses implementing behavior differences. The Scheduler interface defines how calls are distributed, with flexibility to support different algorithms.

My Implementation: I implemented the Penthouse Elevator, Standard Elevator, Simulator core loop, the Simple Scheduler, and the Scheduler interface. I used test-driven development throughout, writing unit tests to validate elevator behaviors, scheduler decisions, and simulation flow. This helped ensure correctness and accuracy through the project.

What I Learned: I got a stronger understanding of Java OOP, SOLID design principles, and TDD. I also practiced working in a collaborative environment using GitHub for version control and handling pull requests.

Card Game Emulator

I designed and implemented the RummyGameBehavior and RummyPlayerBehavior modules of a reusable card game engine. This project allowed me to explore advanced object-oriented design while applying test-driven development principles throughout.

The engine supports multiple card games with modular rulesets. Each game plugs into a shared architecture that includes players, decks, and game state tracking. For Rummy, I wrote logic for handling drawing, discarding, scoring, meld detection, and deck replenishment from the discard pile.

My implementation separates game rules from player decision-making: RummyGameBehavior controls the game loop, turn management, and endgame scoring, while RummyPlayerBehavior handles play decisions like whether to draw from the discard pile, forming melds, laying off cards, and discarding intelligently.

I followed TDD by writing comprehensive unit tests for score calculation, meld validation, and player decision logic using Mockito for mocks. My code emphasizes testability, interface-driven design, and scalability for additional games.

What I Learned: Clean interface-driven architecture, how to balance behavior across game and player layers, and effective use of Java's collections and stream APIs for game state analysis. With this, the project taught me a lot on how to really work as a team through GitHub and how to balance working through writing the code without having everyone else's code to test on.
